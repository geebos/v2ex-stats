# 待办开发任务

## 任务一：优化 XPath 配置逻辑，支持远程获取与本地 fallback

### 1. 需求概述

当前 XPath 选择器硬编码在 `ui/post.ui.ts` 等文件的常量中（如 `V2EX_POST_ALL_CELL_XPATH`、`V2EX_HOT_TOPICS_SHOWN_CELL_XPATH` 等）。V2EX 页面结构可能随版本更新发生变化，导致扩展功能失效。需要将 XPath 配置外置，支持从 GitHub 仓库远程获取最新配置，同时保留本地默认配置作为 fallback，保证在网络不可用时扩展依然正常运行。

### 2. 功能点拆分与依赖

| 功能点序号 | 功能点名称 | 功能点描述 | 前置依赖 |
|------------|------------|------------|----------|
| 1 | XPath 配置类型定义 | 定义 `XPathConfig` 接口，包含主页帖子、今日热议、评论区等各场景所需的 XPath 字符串字段；将 `ui/post.ui.ts`、`ui/comments.ui.ts` 中所有硬编码 XPath 常量收拢到该类型中 | 无 |
| 2 | 本地默认配置 | 将现有硬编码 XPath 常量整理为 `defaultXPathConfig` 对象，作为 fallback 兜底数据，存放于 `service/xpath-config.ts` | 1 |
| 3 | 远程配置拉取模块 | 实现 `fetchRemoteXPathConfig()` 函数，从 GitHub raw 文件地址获取最新 JSON 配置，带超时控制（3s）；失败时静默返回 `null` | 2 |
| 4 | 本地缓存层 | 将成功拉取的远程配置缓存到 `local:xpathConfig`（storage），记录最后更新时间；读取时优先返回缓存，缓存超过 24 小时才重新请求 | 3 |
| 5 | 配置加载入口 | 实现 `getXPathConfig()` 函数：优先读取有效缓存 → 尝试远程拉取并缓存 → 兜底返回本地默认配置；整个过程保证同步返回（首次加载前先 await） | 2, 3, 4 |
| 6 | 业务层替换 | 将 `ui/post.ui.ts`、`ui/comments.ui.ts` 中所有硬编码 XPath 常量替换为从 `getXPathConfig()` 读取；在页面初始化时提前 await 一次配置加载，后续复用 | 5 |

## 任务二：优化忽略 / 恢复功能的操作 UI 逻辑

### 1. 需求概述

当前忽略 / 恢复按钮的初始化（`initPostIngoreButtons`、`initPostRecoverButtons`、`initHotTopicIgnoreButtons`、`initHotTopicRecoverButtons`）与帖子的过滤渲染（`removeIgnoredPosts`、`removeIgnoredPostsHot`）是分开执行的，且按钮初始化依赖 2000ms 延迟的 `loadedObserver`，存在时序不稳定、操作后 UI 状态不实时同步等问题。需要重构为：按钮随帖子渲染时一并注入，操作结果实时反映在 DOM 和计数上，交互更清晰流畅。

### 2. 功能点拆分与依赖

| 功能点序号 | 功能点名称 | 功能点描述 | 前置依赖 |
|------------|------------|------------|----------|
| 1 | 合并忽略按钮注入时机 | 将 `initPostIngoreButtons` 的逻辑合并到 `removeIgnoredPosts` 中：在遍历可见帖子时同步为每个帖子注入忽略按钮，消除两步分离带来的时序问题 | 无 |
| 2 | 合并恢复按钮注入时机 | 将 `initPostRecoverButtons` 的逻辑合并到 `removeIgnoredPosts` 中：将已忽略帖子移入隐藏容器后同步注入恢复按钮，保证恢复按钮总是与隐藏帖子同步存在 | 1 |
| 3 | 今日热议按钮注入合并 | 同理，将 `initHotTopicIgnoreButtons` 和 `initHotTopicRecoverButtons` 的逻辑合并到 `removeIgnoredPostsHot` 中，一次遍历完成分类与按钮注入 | 无 |
| 4 | 切换按钮计数实时更新 | 忽略 / 恢复操作触发后，实时更新切换按钮（`.v-stats-toggle-bar`）上显示的忽略帖子数量，避免计数与实际不符 | 1, 2, 3 |
| 5 | 隐藏容器自动显示 / 隐藏 | 当忽略帖子数量降为 0 时自动隐藏切换按钮和隐藏容器；从 0 增加到 1 时自动创建并插入；简化边界状态处理 | 4 |
| 6 | 移除冗余的延迟初始化逻辑 | 在 `ui/index.ts` 的 `registerMutationObserver` 中移除对四个独立 `init*Buttons` 函数的调用，由合并后的渲染函数统一负责按钮注入 | 1, 2, 3 |
